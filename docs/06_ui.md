# UI設計

## デザイン原則

- **Clean & Modern**: 余白を適切に取り、情報を詰め込みすぎない。shadcn/ui のデフォルトスタイルをベースに、洗練された業務用アプリケーションを目指す。
- **Predictable**: ユーザーが直感的に操作できるよう、一般的な Web アプリケーションの UI パターンに従う。
- **Feedback**: ユーザーの操作（保存、削除、読み込み）に対して、トースト通知やローディングスピナーで明確なフィードバックを返す。

## コンポーネント責務分離

UI コンポーネントは以下のディレクトリ構成で責務を分離します。

1. **`components/ui`**: `shadcn/ui` で生成されるプリミティブなコンポーネント（Button, Input, Tableなど）。ビジネスロジックを持たない。
2. **`features/*/components`**: ドメイン固有のコンポーネント（ClubCard, PlayerFormなど）。Relay の Fragment を持ち、データ要件を宣言する場合がある。
3. **`app/*` (Pages)**: ルーティングとデータフェッチのルートとなるページコンポーネント。`Suspense` バウンダリの設定や、Relay Query の発行を行う。

## テーマ設計

- **Tailwind CSS v4** の Variables 機能を利用し、`:root` (ライトモード) と `.dark` (ダークモード) で色変数を定義・切り替え可能にする。
- メインカラー、背景色、テキスト色を一元管理し、コンポーネント間で一貫性を保つ。

## フォーム設計方針

- **ライブラリ**: `react-hook-form` と `zod` (推奨) を組み合わせ、堅牢なバリデーションを実装する。
- **エラー表示**: 入力フィールドの直下にインラインでバリデーションエラーを表示する。
- **送信状態**: 送信中はボタンを非活性化（Disabled）し、スピナーを表示して二重送信を防止する。
- **初期値**: 編集時は既存データをフォームの初期値（Default Values）として設定する。
